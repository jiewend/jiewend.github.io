从入门到发布网址的个人博客搭建教程

把大象放进冰箱需要三步：打开冰箱，把大象放进去，关上冰箱
博客也是三步：前期准备，编写博客，上传网页

前期准备

  • 学习markdown编辑
    Markdown 是一种轻量级标记语言，创始人为约翰·格鲁伯（John Gruber）。它允许人们“使用易读易写的纯文本格式编写文档，然后转换成有效的XHTML(或者HTML)文档”。这种语言吸收了很多在电子邮件中已有的纯文本标记的特性。
    本教程的博客编写是基于markdown的，没接触过markdown的同学可以移步markdown指南
  • 安装node.js，下载地址为node.js
    Node.js是一个基于Chrome V8 引擎的 JavaScript 运行时。 Node.js 使用高效、轻量级的事件驱动、非阻塞 I/O 模型。Node.js 的生态系统是目前最大的开源包管理系统。看不懂？没关系。你需要知道的是：在这里它就是一个把你写的博客转换成网页的软件就可以了。

  • 一个github的账号，注册地址为github.com
    github是一个通过Git进行版本控制的软件源代码托管服务。不明白github是什么？，无所谓。你知道这是用来托管你的博客网页的就可以了，博客搭建完成后的效果是可以通过网址 username.github.io 来访问你的博客主页，username为github注册用户名

  • git安装，下载地址为git
    git是一个分布式版本控制软件。不知道是什么？不重要。你需要知道的就是：在这里它就是一个用于把生成的网页上传到github的软件。

以上软件的安装都是先下载，然后一直点next或下一步就好了。

本地编写博客

  • 首先打开node.js
    点开win键，在最近添加里打开“Node.js command prompt”，出现了一个小黑框，用户交互使用用命令行进行。不要慌，命令行也是给人用的。
  • 在打开的黑框里输入命令
    • 创建博客文件夹
      • npm install hexo-cli -g 安装hexo博客框架
      • hexo init username.github.io hexo初始化，创建一个博客文件夹。注意：username要换成准备工作中的github账号用户名
      • cd username.github.io 进入创建的文件夹。同上，username要换成github用户名
      • npm install 在文件夹里下载所需文件，具体是什么我也不知道，这个不重要。
    • 生成网页
      • hexo new test 生成网页上的第一篇博文，博文标题为test。内容为空。
      • pwd 显示当前的文件夹目录，记住这个目录，下文记为“博客目录”。 在资源管理器里打开进入该目录，打开./source/_posts里的test.md文件，打开方式可以是windows自带的记事本。个人推荐MdCharm，下载地址MdCharm。划重点：编辑请务必用MarkDown。
      • 博客写完后就可以记得保存，回到黑色框框输入
        • hexo clean
        • hexo g 生成博客网页
    • 如果前几步都没有问题，那么就可以开始检验你的成果了,在黑色框框输入
      • hexo s,启动本地服务器
      • 打开一个浏览器，在地址栏输入“localhost:4000”。就可以看到你的博客网站啦！！

上传网页到github

  • 在github中创建一个仓库，命名为“username.github.io”。同上，username换成你的github用户名
  • 安装 hexo-deployer-git 在黑色框框里输入npm install hexo-deployer-git –save
  • 在博客目录下打开一个名为 _config.yml 的文件，可以用记事本打开。配置deploy如下

deploy:
type: git
repo:
branch: [branch]

  • 最后一步，在黑色框框输入hexo d， 将网页上传到github上
    • 注意：如果是第一次上传，黑色框会弹出一个登陆界面，输入你的github账户密码即可。上传后如果出现以下字样，表明成功上传了。

INFO Deploy done: git

  • 打开浏览器，输入username.github.io，就可以看到你的博客网页啦！！！

第一次写教程，写得不好，有很多地方都可以多加完善，比如可以插入图片，使教程更容易读懂。比如可以更换网站的主题，比如可以加入博客评论功能等等，有时间再来完善。
